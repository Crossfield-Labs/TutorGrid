## 一、产品定位（一页纸版）

### 1.1 一句话

> **智格生境是一张可以和AI共写的活书桌：你记两句，Copilot帮你整理好；你写一句"帮我跑"，编排引擎就替你做了。**

### 1.2 核心价值观

**发挥你的主观能动性 + 一个和你一样懂的Copilot副驾驶，你俩合力。**

- 你是主驾，AI是副驾
- 你把握方向理解知识，AI负责整理和执行力
- 做任务的过程就是学习的过程
- 回头看文档就知道自己学了什么做了什么

### 1.3 产品三层结构

```
工作区 (Workspace)                    ← 项目级 Hub，可自定义视觉（背景图/主题色）
  ├─ 元数据：name / topBarBg / pageBg
  └─ Hyper 文档 (Split-View)          ← 主战场，左文档右磁贴
       ├─ 左 col=6: 文档共做区        ← TipTap 文档；AI 内容是文档节点（co-author）
       ├─ 右 col=6: 固定 Grid 磁贴工具箱
       └─ 右下角: Chat 浮窗            ← 独立 Chat session，可有多条
                  ├─ session 1
                  ├─ session 2
                  └─ ...（共享当前文档作为上下文）
```

**核心原则**："你和 AI 都是一等作者"：
- **文档内 AI** = 共同作者的产出（嵌入文档节点，跟用户段落同等地位、可编辑/删除/引用）
- **右下角 Chat** = 助理对话（独立时间线，多 session 并存，**不污染**文档）
- 两者**共享上下文**（chat 的 system prompt 自动注入当前文档摘要 + 最近段落），**不共享消息流**
- 跨视图同步走"主动操作"：chat 答案 → 📌 插入到文档；文档 AI 节点 → 💬 在 Chat 中讨论

### 1.4 信息架构与 Sidebar 导航

**数据三层**：

```
Workspace (工作区)
├─ 元数据：id / name / appearance{topBarBg, pageBg, sidebarColor} / createdAt
└─ Hyperdocs[]
   ├─ 元数据：id / title / lastEditedAt
   ├─ TipTap 文档树（含 AI co-author 节点 = 文档结构的一部分，跟随文档保存）
   └─ ChatSessions[]                 ← 一个 Hyperdoc 可有多条独立 chat
      ├─ 元数据：id / title / hyperdocId / createdAt
      ├─ messages[]（独立时间线）
      └─ systemContext = "当前 hyperdoc 摘要 + 最近段落"（动态注入，不持久化在消息里）
```

**Sidebar 结构**（左侧固定栏，复用现有 [navigation.ts](../TutorGridFront/src/configs/navigation.ts)）：

```

├─ 顶部：Logo

```
┌─────────────────────────────────┐
│ [💬 默认] [公式推导] [+ 新建]    │  ← chat session tabs
├─────────────────────────────────┤
│  消息流（绑定当前 hyperdoc 上下文）│

### 2.2 接口规约：Chat SSE

**`POST /api/chat/stream`**

请求体：
```json
{
  "session_id": "sess_xxx",
  "message": "线性回归的损失函数是什么？",
  "course_id": "course_datamining",       // 可选，指定RAG检索的课程
  "tools": ["rag", "tavily"],              // 可选，指定可用Tool
  "context": {                             // 可选，文档上下文
    "doc_id": "hyper_001",
    "recent_paragraphs": ["用户最近写的内容..."]
  }
}
```

SSE响应流（`text/event-stream`）：
```
data: {"type":"start","message_id":"msg_001"}

data: {"type":"tool_call","tool":"rag","query":"线性回归 损失函数"}

data: {"type":"tool_result","tool":"rag","citations":[{"source":"PPT第42页","chunk":"..."}]}

data: {"type":"delta","content":"线性回归"}

data: {"type":"delta","content":"的损失函数通常使用"}

data: {"type":"delta","content":"均方误差（MSE）……"}

data: {"type":"done","message_id":"msg_001","metadata":{"tokens_used":156,"tools_called":["rag"]}}
```

前端消费方式：
```typescript
const response = await fetch('/api/chat/stream', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(payload)
})
const reader = response.body!.getReader()
const decoder = new TextDecoder()
while (true) {
  const { done, value } = await reader.read()
  if (done) break
  const lines = decoder.decode(value).split('\n')
  for (const line of lines) {
    if (line.startsWith('data: ')) {
      const event = JSON.parse(line.slice(6))
      // 按 event.type 分发处理
    }
  }
}
```

### 2.5 数据模型（前后端共用）

V5 拆分两套模型，对应"文档作者产出"与"助理对话"两种语义：

#### 2.5.1 文档内 AI 节点（TipTap node，跟随文档保存）

```typescript
// 嵌入 TipTap 文档树的 AI 节点，是文档结构的一部分
interface DocumentAiNode {
  id: string                    // 节点唯一 ID
  hyperdocId: string            // 所属 Hyperdoc
  type: 'bubble' | 'citation' | 'task_register' | 'task_result' | 'quiz' | 'flashcard'
  // 'bubble' = AI 自动补充的段落（co-author）
  // 'citation' = 用户选中后 AI 给出的引用标注
  // 'task_register' = /task 注册卡（编排任务起点）
  // 'task_result' = 编排任务结果回写块
  // 'quiz'/'flashcard' = AI 产出的交互组件
  content: string               // TipTap 内容（Markdown / JSON）
  metadata?: {
    quotedRange?: { from: number; to: number }  // citation 引用的原文位置
    taskId?: string                              // 关联的编排任务 ID
    sourceChatMessageId?: string                 // 由 Chat "插入到文档"产生时的来源
    citations?: Array<{ source: string; chunk: string; score: number }>
  }
  authorRole: 'ai' | 'user'    // 用户也可以编辑/创建（co-author 嘛）
  createdAt: number
  updatedAt: number
}
```

**关键约束**：
- 这个模型**不存在独立 store**，它就是 TipTap 文档 JSON 树里的节点
- 文档保存 = 这些节点跟着文档持久化
- 文档删除 = 这些节点也没了（不进 chat 历史）

#### 2.5.2 Chat 会话（独立持久化，按 session 分组）

```typescript
interface ChatSession {
  id: string                    // 会话唯一 ID
  hyperdocId: string            // 绑定的 Hyperdoc（chat 必须依附文档）
  title: string                 // 用户起的会话标题（默认"新会话"，AI 可建议）
  createdAt: number
  lastActiveAt: number
  // systemContext 不存数据库；每次发请求时由前端动态注入：
  //   "你正在协助用户编辑《<doc.title>》。当前文档摘要：<summary>。最近段落：<recent>"
}

interface ChatMessage {
  id: string
  sessionId: string             // 所属 ChatSession
  role: 'user' | 'ai' | 'system'
  content: string               // 主内容
  metadata?: {
    citations?: Array<{ source: string; page?: number; chunk: string; score: number }>
    toolUsed?: string[]         // RAG / Tavily 等本次调的 tool
    insertedToDocAsNodeId?: string  // 用户点"📌 插入到文档"后产生的节点 ID
  }
  timestamp: number
}
```

**关键约束**：
- 一个 Hyperdoc 可绑多个 ChatSession（默认创建一个"默认会话"）
- ChatMessage 不写入文档；除非用户主动点"📌 插入到文档"，会复制一份成 `DocumentAiNode`（type:'bubble'）
- 反向：用户在文档里看到 AI bubble → 点"💬 在 Chat 中讨论" → 当前 session 里 append 一条引用消息（含 sourceNodeId）

#### 2.5.3 跨视图同步约定

| 操作 | 来源 | 目标 | 实现 |
|------|------|------|------|
| 📌 插入到文档 | ChatMessage | 新建 DocumentAiNode | 复制内容 + 记录 `sourceChatMessageId` |
| 💬 在 Chat 中讨论 | DocumentAiNode | 当前 ChatSession 加引用消息 | 引用 nodeId + 摘录前 200 字 |
| /ai 改写当前段落 | 用户 slash 命令 | 替换当前 DocumentAiNode | 不入 chat |
| /task 注册编排 | 用户 slash 命令 | 文档内 task_register 节点 + 右侧 TaskTile | 走 WS 编排协议（独立通道）|
| 选中文字 → "AI 评论" | BubbleMenu | 文档内 citation 节点 | 不入 chat |
| 用户在右下角 chat 直接问 | ChatMessage | 当前 ChatSession 时间线 | SSE 流 |

**默认完全独立**，跨视图只在用户主动操作时发生。
